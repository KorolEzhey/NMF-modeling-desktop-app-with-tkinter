# РГР: Выделение независимых признаков (NMF)

Расчётно-графическая работа по теме 7, глава 10 книги Тоби Сегарана «Программируем коллективный разум».

## Описание

Desktop-приложение на Python с графическим интерфейсом (tkinter), реализующее **ручную версию алгоритма неотрицательной матричной факторизации (NMF)** для автоматического выделения скрытых тем из набора текстовых документов без учителя.

## Теоретические основы

### Что такое независимые признаки

**Независимые (латентные) признаки** — это скрытые характеристики данных, которые напрямую не заданы в исходном наборе, но выявляются алгоритмически. В отличие от кластеризации, где каждый объект попадает ровно в одну группу, метод выделения признаков строит каждый объект как **комбинацию нескольких базовых признаков** с различными весами.

Аналогия из книги Сегарана: задача о вечеринке. В комнате разговаривает много людей одновременно — это «шум». Человеческий мозг умеет вычленять отдельные голоса (независимые признаки) из какофонии. NMF делает то же самое для текстов: находит «голоса» (темы), из которых состоят документы.

### Неотрицательная матричная факторизация (NMF)

Алгоритм факторизует исходную матрицу `V` на произведение двух меньших матриц:

```
V ≈ W × H
```

| Матрица | Размерность | Содержимое |
|---|---|---|
| **V** | `m × n` | Исходная матрица «документы × слова». `V[i,j]` = сколько раз слово `j` встретилось в документе `i`. |
| **W** | `m × k` | Матрица весов «документы × темы`. `W[i,t]` = насколько тема `t` присутствует в документе `i`. |
| **H** | `k × n` | Матрица признаков «темы × слова`. `H[t,j]` = насколько слово `j` характерно для темы `t`. |

Здесь `m` — число документов, `n` — число уникальных слов, `k` — число тем (параметр `pc`, выбирает пользователь).

### Ключевое ограничение: неотрицательность

Все элементы `W` и `H` ≥ 0. Это означает:
- Темы складываются, но не вычитаются друг из друга.
- Результаты легко интерпретировать: высокий вес слова в теме = слово характерно для этой темы.
- Нельзя получить «отрицательную тему», что делает NMF устойчивым и понятным.

### Целевая функция

Алгоритм минимизирует **сумму квадратов разностей** между исходной матрицей и её реконструкцией:

```
cost = Σ (V[i,j] − (W×H)[i,j])²
```

Функция `difcost()` в `nmf_core.py` реализует именно эту формулу. Чем меньше `cost`, тем точнее матрицы `W` и `H` описывают исходные данные.

### Мультипликативные правила обновления

На каждой итерации матрицы `W` и `H` обновляются по правилам из оригинальной статьи Lee & Seung (1999), приведённым в книге Сегарана:

```
H_new = H × (Wᵀ × V) / (Wᵀ × W × H)
W_new = W × (V × Hᵀ) / (W × H × Hᵀ)
```

Все операции — поэлементные. Начальные значения `W` и `H` случайны, и с каждой итерацией ошибка `cost` уменьшается.

## Как работает программа: пошагово

### Шаг 1. Загрузка и предобработка текстов

Исходные данные — 70 коротких новостных сниппетов из `data.py`. Каждый текст проходит через функцию `separatewords()`:

1. **Токенизация**: текст разбивается на слова с помощью регулярного выражения.
2. **Очистка**: удаляются HTML-теги (если есть), пунктуация, цифры.
3. **Лемматизация**: `pymorphy3` приводит слова к начальной форме. Например:
   - *выиграла* → **выиграть**
   - *акции* → **акция**
   - *бюджета* → **бюджет**
4. **Фильтрация стоп-слов**: удаляются предлоги, союзы, местоимения (*в, на, и, что, этот* и др.) — около 100 слов.
5. **Фильтрация по частоте**: остаются только слова, встречающиеся **минимум в 2 документах**, но **не более чем в 60 %** документов. Это убирает слишком редкие термины (бесполезные для тем) и слишком общие (не различают темы).

Результат: матрица `V` размером `70 × 57` (70 документов, 57 значимых слов).

### Шаг 2. Инициализация матриц W и H

Матрицы `W` (70 × k) и `H` (k × 57) заполняются случайными числами из диапазона [0, 1]. Параметр `k` (количество признаков) задаёт пользователь в интерфейсе (по умолчанию 4).

### Шаг 3. Итеративная факторизация

Функция `factorize(v, pc, iter)` выполняет заданное число итераций:

1. Вычисляет текущую реконструкцию: `WH = W × H`
2. Считает ошибку: `cost = difcost(V, WH)`
3. Обновляет `H` по мультипликативному правилу
4. Обновляет `W` по мультипликативному правилу
5. Повторяет

Если `cost` становится равным 0 (практически невозможно на реальных данных), цикл прерывается.

### Шаг 4. Интерпретация результатов

После завершения итераций программа анализирует матрицы:

- **Из H (признаки)**: для каждой темы `t` выбираются слова с наибольшими значениями `H[t,j]`. Это «ключевые слова темы».
- **Из W (веса)**: для каждой темы `t` выбираются документы с наибольшими значениями `W[i,t]`. Это «наиболее релевантные документы».
- **Обратная связь**: для каждого документа `i` выбираются темы с наибольшими `W[i,t]`. Это показывает, из каких тем состоит документ.

### Шаг 5. Визуализация и сохранение

В графическом интерфейсе результаты отображаются в четырёх вкладках:

| Вкладка | Содержимое |
|---|---|
| **Темы (признаки)** | Ключевые слова каждой темы + топ-5 релевантных документов с числовыми весами |
| **Документы и темы** | Для каждого документа — топ-3 темы с весами |
| **График весов** | Горизонтальная столбчатая диаграмма топ-10 слов первой темы |
| **Матрица** | Размерности V, W, H и фрагмент исходной матрицы (первые 5 документов × 10 слов) |

По кнопке **«Сохранить отчёт»** создаются два файла:
- `features.txt` — аналог функции `showfeatures()` из книги
- `articles.txt` — аналог функции `showarticles()` из книги

## Пример работы

### Исходные данные

70 документов, 4 тематические группы.

### Параметры запуска

- Количество признаков: **4**
- Итераций: **100**

### Результаты

**Итоговая ошибка реконструкции:** `98,47`

> Это нормально для NMF на реальных данных: матрица `70 × 57` не может быть точно воспроизведена произведением `70 × 4` и `4 × 57`. Увеличение числа тем уменьшает ошибку, но при `pc = 70` каждый документ получит свою тему — переобучение.

**Выделенные темы:**

| Тема | Ключевые слова | Интерпретация |
|---|---|---|
| **Тема 1** | победа, одержать, благодаря, этап, мировой, рекордный, контракт, футболист | Спорт (общий): победы, рекорды, достижения |
| **Тема 2** | метр, дистанция, сто, финал, чемпионат, турнир, выиграть, сборная | Спорт (соревнования): дистанции, финалы, чемпионаты |
| **Тема 3** | партия, объявить, избирательный, начало, закон, выступить, министр, старт | Политика (выборы и законы): партии, законы, министры |
| **Тема 4** | провести, правительство, требование, бюджет, обсудить, выборы, социальный, столица | Политика (правительство): совещания, бюджеты, требования |

**Пример распределения тем в документе:**

Документ *«Футболист забил красивый гол в концовке матча»*:
- Тема 2 (спортивные соревнования): **вес 0,74**
- Тема 1 (спортивные достижения): **вес 0,66**
- Тема 3 (политика): **вес 0,00**

Документ содержит смесь двух спортивных признаков, но не содержит политических — точно как в задаче о вечеринке: в одном месте слышны два голоса, а третий отсутствует.

## Сравнение с кластеризацией

| | Кластеризация (глава 3) | NMF (глава 10) |
|---|---|---|
| **Принцип** | Каждый объект → ровно одна группа | Каждый объект = комбинация нескольких тем |
| **Границы** | Жёсткие | Мягкие (веса) |
| **Интерпретация группы** | Среднее членов кластера | Набор ключевых слов (признак) |
| **Применимость к текстам** | Плохо: новости бывают о нескольких темах | Хорошо: смесь тем с разными весами |

## Описание интерфейса

При запуске открывается окно «Выделение независимых признаков (NMF)».

### Панель управления (верх)

- **Поле «Количество признаков»** — spinbox, значение по умолчанию `4` (диапазон 2–10).
- **Поле «Итераций»** — spinbox, значение по умолчанию `100` (диапазон 10–500).
- **Кнопка «Выделить признаки»** — запускает полный пайплайн.
- **Кнопка «Сохранить отчёт»** — экспортирует `features.txt` и `articles.txt` (активна после запуска).

### Левая панель «Исходные тексты»

- **Список заголовков** — 70 кратких названий документов. При выборе элемента в окне предпросмотра появляется полный текст.
- **Окно предпросмотра** — полный текст выбранной новости (только для чтения).

### Правая часть — вкладки с результатами

- **«Темы (признаки)»** — для каждой темы: 8 ключевых слов и 5 самых релевантных документов с числовыми весами.
- **«Документы и темы»** — для каждого документа: топ-3 темы с весами.
- **«График весов»** — столбчатая диаграмма топ-10 слов для первой темы.
- **«Матрица»** — размерности матриц и фрагмент `V` для проверки корректности.
